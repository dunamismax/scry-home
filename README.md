# scry-home

Control plane monorepo for local repo operations, encrypted backup workflows, CAB packet scaffolding, and an in-app operator dashboard.

## Stack

- Runtime: Bun
- Workspace / package manager: Bun
- Language: TypeScript
- Frontend: TanStack Start + Router + Query
- Database: PostgreSQL + Drizzle ORM
- Validation: Zod
- Auth: Better Auth
- Observability: OpenTelemetry
- Lint / format: Biome
- Tests: Vitest

## Workspace Layout

- `apps/web` — TanStack Start app for repo and backup views
- `packages/core` — shared TypeScript domain utilities and contracts
- `packages/cab` — CAB packet templates and scaffolding
- `packages/db` — Drizzle schema and PostgreSQL client
- `packages/auth` — Better Auth integration
- `packages/observability` — OpenTelemetry bootstrap
- `packages/cli` — Bun CLI for local operations
- `vault` — encrypted backup artifacts only

## Commands

```bash
bun install
bun run dev
bun run check
bun run cli doctor
bun run cli projects:list
bun run cli cab:new --project=scry-home --packet=repo-control-plane-slice
bun run cli setup:config_backup
bun run cli verify:config_backup
```

## Environment

- `DATABASE_URL`
- `BETTER_AUTH_SECRET`
- `BETTER_AUTH_URL`
- `SCRY_CONFIG_BACKUP_PASSPHRASE`
- `SCRY_SSH_BACKUP_PASSPHRASE`

## Notes

- `apps/web/src/routeTree.gen.ts` is generated by TanStack Router.
- Do not commit decrypted material into `vault/`.
- Use `bun` for workspace commands and package-local scripts.
