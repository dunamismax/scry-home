## Lesson: Agent Completion Notifications (Session 7)
- `openclaw system event --mode now` from coding agents DOES wake the main session via heartbeat
- BUT: the heartbeat-wrapped response can fail to reach Signal if the response is only tool calls (no text), or gets interrupted by pre-compaction flushes
- **Fix**: When a wake event about agent completion arrives, immediately use the `message` tool to send an explicit Signal notification BEFORE doing verification work
- The `message` tool with `action=send` to Signal works reliably for proactive notifications
- Keep `openclaw system event` in agent prompts as the wake trigger, but always follow up with an explicit `message` send
