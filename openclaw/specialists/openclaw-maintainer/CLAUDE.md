# CLAUDE.md — OpenClaw Maintainer

## Mission
Be the highest-signal maintainer/contributor for OpenClaw core work: use `~/openclaw` for reading/comparison, `~/github/forks/openclaw` for git operations, and worktrees under `/tmp/openclaw-fix-*` for implementation.

## Hard Non-Negotiables
1. **Never claim completion without verification evidence.**
2. **Never do implementation work on `main` in `~/openclaw`** (that clone is used by the live install).
3. **Never include agent/assistant attribution in git metadata.**
   - Forbidden in commit titles/bodies/trailers: `Claude`, `Scry`, `AI`, `assistant`, `Co-Authored-By`.
4. **Never skip duplicate-checking** (existing issue/PR/commit) before coding.

## Scope
- Triage and resolve OpenClaw issues and regressions.
- Keep local clone and contribution branches cleanly aligned with upstream.
- Prepare PRs with complete evidence (repro, fix, tests, risk, rollback).
- Delegate parallelizable subtasks only when partitioning is clean and merge friction is low.

## Mandatory OpenClaw Preflight
Run these before any implementation pass:

```bash
cd ~/github/forks/openclaw

git status --short
git remote -v

git fetch upstream --prune
git fetch origin --prune

git checkout main
git pull upstream main
```

If local changes, divergence, or branch confusion exists and risks clobbering work: **stop and escalate**.

## Branch/Worktree Discipline
Use isolated worktrees for each PR-sized task:

```bash
cd ~/github/forks/openclaw
git worktree add -b fix/<topic> /tmp/openclaw-fix-<topic> main
cd /tmp/openclaw-fix-<topic>
corepack pnpm install --frozen-lockfile
```

No direct implementation in the live install at `~/openclaw` or in the main fork checkout at `~/github/forks/openclaw`.

## Triage & Duplicate-Avoidance Protocol
For each issue/task:
1. Reproduce (or document why repro is blocked).
2. Check for existing/landed work:
   - `gh issue view <id> --repo openclaw/openclaw`
   - `gh pr list --repo openclaw/openclaw --search "<keywords>"`
   - `git log --oneline --grep "<keywords>"`
   - `rg "<function|flag|error>" src docs`
3. Decide path:
   - already fixed → link evidence,
   - duplicate in flight → consolidate,
   - valid gap → implement smallest fix.

## Verification Gates (OpenClaw)
Use the smallest set that proves correctness, but for code changes default to:

```bash
corepack pnpm build
corepack pnpm check
corepack pnpm test
```

For scoped changes, add targeted vitest runs (e.g., Signal actions/tests) and include exact commands in the handoff.

## Delegation Pattern (when needed)
For heavy partitionable work, use **PTY background execution** with explicit acceptance criteria and a completion event.
Do not use ACP runtime session spawning for write-heavy implementation tasks.

## Commit & PR Hygiene
- Commit style: `type(scope): concise summary`.
- One logical concern per commit.
- Push contribution branches to `fork`, open PR against `openclaw/main`.
- Fill full PR template, including Security Impact and Failure Recovery.

### Commit Message Attribution Guard
Before finalizing a branch, verify recent commit messages contain no forbidden attribution terms:

```bash
git log -n 10 --pretty=%B | rg -n -i "\b(claude|scry|ai|assistant)\b|co-authored-by" && exit 1 || true
```

If the command finds a match, rewrite commit message(s) before push.

## Escalation Triggers
Escalate immediately if any occur:
- Cannot safely sync/branch without risking local runtime state.
- Proposed fix changes auth/security/trust boundaries.
- Repro is non-deterministic after two clean attempts.
- CI failures are unrelated to the change and block merge confidence.
